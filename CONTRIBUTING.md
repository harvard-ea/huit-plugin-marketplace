# Contributing

This repo is a **data store**. Adding or changing an MCP server is done through
the **submission MCP server** (`harvard-huit/huit-plugin-marketplace-submit`),
which validates the submission, generates the plugin files, and opens a PR here.

## Add a server (recommended)

From your MCP client (Claude Code / Codex), with the submission server connected
and after authenticating with Okta:

1. Ask to *add an MCP server to the marketplace*, providing the server's name,
   description, HTTPS URL, auth type, and a few tags.
2. The submission server validates it (and best-effort discovers its tools),
   then opens a PR to this repo.
3. A maintainer reviews and merges. The catalogs and listing update as part of
   the PR; GitHub Pages redeploys from `main`.

## Manual PRs

You can still open a PR by hand, but note there is **no build tooling in this
repo** — generated files (plugin manifests, catalogs, `docs/`) are produced by
the submission server. A hand-authored PR that only adds a `server.yaml` will be
regenerated by the submission server (or by a maintainer) before merge.

## Notes

- Credentials are never stored here. Auth-requiring servers ship a
  `<YOUR_API_KEY>` placeholder each user replaces locally.
- For private/internal repos, your GitHub token/SSH key must be **SSO-authorized**
  to the `harvard-huit` org.
