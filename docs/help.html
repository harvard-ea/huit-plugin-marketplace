The HUIT Plugin Marketplace distributes Model Context Protocol (MCP) servers as one-click plugins for Claude Code and (soon) OpenAI Codex. This page covers installing the marketplace and plugins, and submitting your own server.

Install in Claude Code

  1. Add the marketplace: 
    /plugin marketplace add harvard-ea/huit-plugin-marketplace
  2. Install a plugin (browse the marketplace for names): 
    /plugin install <plugin-name>@huit-plugin-marketplace
  3. Apply it: 
    /reload-plugins
  4. If the server requires authentication, run /mcp and complete the sign-in flow when prompted.

Submit a server for review

Submissions are made through the submission MCP server, which validates your server and opens a pull request to the marketplace for a maintainer to review. The easiest path is the companion plugin:

  1. Add the tools marketplace and install the submission plugin: 
    /plugin marketplace add harvard-huit/huit-plugin-marketplace-submit
/plugin install huit-marketplace-submit@huit-plugin-marketplace-tools
/reload-plugins
  2. Authenticate the huit-plugin-submission MCP server (/mcp).
  3. Ask Claude to “submit my MCP server to the marketplace.” The guided skill collects the details (name, description, URL, tags, homepage), checks for name collisions, dry-run validates, and opens the PR.

To change a server that's already published, update it (rather than re-adding): the submission flow opens a PR that replaces the existing entry.

Prefer to drive it yourself? The server exposes list_servers, validate_server (dry run), submit_server (add), and update_server (modify an existing entry) directly.

Team-specific marketplaces

Beyond the public marketplace, teams can have their own curated marketplace (a separate destination tier). First-pass guidance:

  • Each team marketplace is its own data repository with its own catalog and access controls.
  • Submitters choose a destination tier when submitting; access is gated by team membership (authorized via the submitter's identity).
  • Setup and policy are coordinated with HUIT — this section will expand with concrete steps as team marketplaces roll out.
